# План разработки RBAC и User Management

Внедрение полноценного управления пользователями и политиками безопасности разделено на следующие этапы:

## 🟢 Горизонт 0: Текущее состояние (MVP v1.1)
- [x] Описана доменная модель `User`, `Permission`, `RedactionRule`, `RetentionPolicy`.
- [x] Ролевая модель интегрирована в API и UI (сокрытие кнопок для `Observer`).
- [x] Базовое логирование (Audit) действий по патчам.

---

## 🟡 Горизонт 1: Enterprise Auth & Core RBAC
*Приоритет: Безопасный доступ и управляемость.*
- [ ] **SSO Integration:** Внедрение Passport.js (или NextAuth) для интеграции с OIDC/SAML провайдерами.
- [ ] **JIT Provisioning:** Автоматическое создание `User` при первом входе через SSO, маппинг групп из токена в `UserRole`.
- [ ] **RBAC API:** Разработка эндпоинтов для изменения ролей пользователей (для Admin).
- [ ] **Interactive Debug:** Итеративная отладка SSO-флоу в диалоге с разработчиком (симуляция ответов IdP).

---

## 🔵 Горизонт 2: Data Security & Admin UI
*Приоритет: Визуальное управление политиками и защита данных.*
- [ ] **Admin Dashboard UI:** Разработка нового раздела в `demo-shell` для управления пользователями и просмотра системной статистики.
- [ ] **Redaction Engine:** Внедрение regex-парсера, который будет динамически применять `RedactionRule` к тексту из `SourcePanel` на лету (или при сохранении).
- [ ] **Retention Cron:** Создание фонового воркера для отработки `RetentionPolicy` (удаление/архивация старых Workspace).
- [ ] **Collaborative Debug:** Совместная отладка механизмов скрытия данных (Redaction) для обеспечения отсутствия утечек PII.

---

## 🟣 Горизонт 3: Advanced Audit & Compliance
*Приоритет: Готовность к сертификациям ИБ.*
- [ ] **Audit Trail UI:** Добавление вкладки "Audit Log" в Workspace для отображения полного таймлайна событий.
- [ ] **SIEM Webhooks:** Разработка механизма экспорта `AuditRecord` во внешние системы безопасности (Splunk).
- [ ] **Granular Permissions:** Переход от жестких ролей к динамическим группам с настраиваемыми `Permissions` (Custom Roles).
- [ ] **Ecosystem Debug:** Финальное тестирование матриц доступа и аудит-логов в диалоге с разработчиком.
