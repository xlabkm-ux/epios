# 👮 Регламент работы администраторов (Functioning Beta)

Данный документ определяет права, обязанности и порядок действий **Администратора системы** в рамках функционирующей Beta-версии Epistemic OS.

## 1. Роль и Манифест
Администратор — хранитель структурной и семантической целостности системы. Работа ведется по **Спиральной модели**: от настройки ядра (организации) к обеспечению работы периферии (рабочих мест).

### Ключевые принципы:
- **Zero-Unregistered:** Запрещен доступ к любым корпоративным ресурсам без регистрации.
- **WP-Isolation:** Каждое Рабочее место (WP) должно быть изолировано и обеспечено квотами.
- **Semantic Guard:** Администратор обязан следить за единообразием терминологии во всех назначаемых РМ.

## 2. Основные обязанности

### А. Управление оргструктурой (Enterprise)
1. **Реестр Должностей:** Ведение справочника `org_positions`.
2. **Оргструктура:** Управление деревом подразделений `org_units`.

### Б. Ведение Таблицы назначений (WP Management)
Администратор — единственный субъект, имеющий право редактировать Таблицу назначений.
1. Создание новых **Рабочих мест (WP)**.
2. Назначение пользователям связки `(Подразделение + Должность + Роль + Рабочее пространство)`.
3. Мониторинг лимитов и квот для каждого WP.

## 3. Назначение Рабочего места (Provisioning)
Выделение ресурсов и прав производится Администратором через **Таблицу назначений**.
1. Администратор создает РМ (WP).
2. Назначает Владельца (Owner) и других участников.
3. Указывает доступные в рамках этого WP Рабочие пространства (WS).
4. Управляет участниками: Владелец РМ самостоятельно добавляет других зарегистрированных пользователей и назначает им роли (`Reviewer`, `Contributor`, `Observer`).
