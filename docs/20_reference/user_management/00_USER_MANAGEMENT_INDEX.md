# Управление пользователями и Enterprise RBAC: Индекс раздела

**Документ:** Устав раздела управления пользователями и политиками доступа  
**Статус:** Активен (MVP реализован частично, фокус на Enterprise)  
**Цель:** Сбор всей информации и проектирование полноценной подсистемы управления пользователями (Identity), контроля доступа (RBAC), безопасности данных (Redaction) и аудита.

## 📋 Требования к пакету документов по управлению ресурсами
Для корпоративной системы (Enterprise) документация по пользователям обязана регламентировать:
1.  **Формальную модель доступа:** Четкую матрицу связей "Субъект (User/Role) -> Действие (Permission) -> Ресурс (Workspace/Patch/Node)".
2.  **Жизненный цикл учетных записей (Provisioning):** Регламент создания, блокировки и миграции пользователей (вкл. SSO/SAML).
3.  **Политики конфиденциальности данных (Data Security):** Регламент маскирования чувствительных данных (PII/Redaction) и сроков хранения (Retention).
4.  **Аудит и комплаенс:** Требования к непрерывному журналированию критичных действий (Audit Trails) для прохождения проверок ИБ.
5.  **Требования к интерфейсам администратора (Admin UI):** Панели для управления политиками без вмешательства в код.

## 📁 Структура раздела

В соответствии с этими требованиями, раздел разбит на 5 ключевых документов:

### 1. [01_IDENTITY_AND_ACCESS_MODEL.md](./01_IDENTITY_AND_ACCESS_MODEL.md)
Базовая доменная модель (`User`, `UserRole`, `Permission`) и матрица доступа (Access Control Matrix) для ролей `admin`, `reviewer`, `observer`.

### 2. [02_PROVISIONING_AND_LIFECYCLE.md](./02_PROVISIONING_AND_LIFECYCLE.md)
Архитектура регистрации, интеграция с корпоративным SSO (OIDC/SAML) и процессы деактивации (Offboarding).

### 3. [03_DATA_SECURITY_AND_RETENTION.md](./03_DATA_SECURITY_AND_RETENTION.md)
Политики скрытия данных (`RedactionRule`) и жизненный цикл контента (`RetentionPolicy` — архивирование и удаление).

### 4. [04_AUDIT_AND_COMPLIANCE.md](./04_AUDIT_AND_COMPLIANCE.md)
Архитектура структуры `AuditRecord` и механизмы экспорта журналов для SIEM-систем.

### 5. [05_RBAC_DEVELOPMENT_PLAN.md](./05_RBAC_DEVELOPMENT_PLAN.md)
Roadmap реализации недостающих административных интерфейсов (RBAC Admin UI) и доработок бэкенда.
