# Жизненный цикл пользователя и Provisioning

В системе Epistemic OS управление доступом базируется на обязательной идентификации и четко определенных этапах жизненного цикла.

## 1. Обязательная Регистрация (Mandatory Registration)

Любой доступ к ресурсам системы (кроме публичных справочных страниц) требует предварительной регистрации.
- **Саморегистрация:** Пользователь может создать аккаунт самостоятельно (если разрешено политикой предприятия).
- **Регистрация Администратором:** Администратор может создать учетную запись и выслать приглашение.
## 1. Регистрация и Вход (AuthScreen)

В Epistemic OS введена обязательная регистрация через единый интерфейс **AuthScreen**.
- **Регистрация:** Пользователь вводит UID, имя, email и пароль. По умолчанию аккаунт получает статус `Guest/Trainee`.
- **Вход:** Аутентификация по email/password или через OAuth (GitHub).
- **Default Access:** До момента назначения Администратором первого **Рабочего места (WP)**, пользователю доступны только:
  - Демо-режим (ReadOnly).
  - База знаний и инструкции пользователя.
  - ИИ Помощник-Тренер.

## 2. Назначение Рабочего места (Provisioning)
Выделение ресурсов и прав производится Администратором через **Таблицу назначений**.
1. Администратор создает РМ (WP).
2. Назначает Владельца (Owner) и других участников.
3. Указывает доступные в рамках этого WP Рабочие пространства (WS).
или нескольких пользователей **Владельцами (Owner)** этого РМ.
3. **Управление участниками:** Владелец РМ самостоятельно добавляет других зарегистрированных пользователей и назначает им роли (`Reviewer`, `Contributor`, `Observer`).

## 4. Offboarding (Деактивация)

Удаление пользователя в системах Governance запрещено для сохранения целостности Audit Trail.
- **Soft Delete:** Поле `isActive` переводится в `false`.
- **Историческая память:** Все решения (Approve, Propose), принятые пользователем, сохраняются в истории с указанием его UID.
