# 🛠️ Спецификация Административной Панели (Admin Panel Specs)

Панель администрирования — это защищенный раздел внутри модального окна "Настройка", доступный только пользователям с `role === "admin"`.

## 1. Структура Панели (Закладки)

Административная панель содержит следующие специализированные подразделы:

### А. Управление пользователями (User Manager)
- Список всех пользователей системы.
- Фильтры по статусу (`Active`, `Blocked`), роли и департаменту.
- Кнопки быстрого действия: `Edit Role`, `Reset 2FA`, `Terminate Sessions`.

### Б. Мониторинг Ресурсов (Workspace Audit)
- Список всех созданных Workspace.
- Статистика использования места и количества узлов графа.
- Возможность принудительного изменения статуса (например, извлечение из архива).

### В. Безопасность и Политики (Global Policies)
- Редактор глобальных `RedactionRules`.
- Настройка `RetentionPolicies` (время жизни данных).

### Г. Журнал Аудита (System Audit Log)
- Необработанный поток событий безопасности.
- Поиск по ActorUID и типу действия.

## 2. Требования к Интерфейсу

- **Усиленная авторизация:** Переход в раздел Администрации может требовать повторного ввода пароля или 2FA кода (Re-auth).
- **Визуальное отличие:** Раздел должен иметь четкое цветовое обозначение (например, акцент на `var(--accent-red)` или `var(--accent-amber)`), сигнализирующее о повышенной ответственности.
- **Подтверждение действий:** Любое разрушающее действие (удаление, смена роли) требует подтверждения в отдельном диалоговом окне.
