# User Workflow Roadmap: Индекс раздела

**Документ:** Организационный устав раздела User Workflow  
**Статус:** Активен  
**Цель:** Описание структуры и назначения директории `07_user_workflow_roadmap`, которая является центром знаний о путях пользователя (User Journeys), ролевых моделях и сценариях использования платформы Epistemic OS.

## 📁 Структура раздела

Раздел разбит на 4 ключевых документа, описывающих роли, текущие и будущие рабочие процессы:

### 1. [01_CORE_ROLES_AND_PERSONAS.md](./01_CORE_ROLES_AND_PERSONAS.md)
**Что внутри:** Описание ролевой модели (RBAC) с точки зрения продуктовых персон.
- Права и обязанности Author (Автора), Reviewer (Ревьюера), Observer (Наблюдателя) и Admin.
- Как UI адаптируется под каждую из ролей.

### 2. [02_ADR_REVIEW_WORKFLOW.md](./02_ADR_REVIEW_WORKFLOW.md)
**Что внутри:** Детальное описание каноничного рабочего процесса MVP v1.1 (ADR Review).
- Пошаговый "Happy Path" (от загрузки черновика до генерации финального артефакта).
- Взаимодействие человека и AI (асинхронный маппинг, оценка источников).

### 3. [03_GOVERNANCE_AND_PATCHING.md](./03_GOVERNANCE_AND_PATCHING.md)
**Что внутри:** Глубокое погружение в процесс внесения изменений.
- Жизненный цикл `ArtifactPatch` (Предложение -> Голосование -> Применение -> Финализация).
- Роль Readiness Assessment (Оценки готовности) в принятии решений.

### 4. [04_FUTURE_WORKFLOWS_PLAN.md](./04_FUTURE_WORKFLOWS_PLAN.md)
**Что внутри:** План развития пользовательских процессов за рамками MVP (Roadmap).
- Синхронная и асинхронная коллаборация нескольких ревьюеров.
- Интеграция с внешними Workflow (GitHub PR, Jira).
- Развитие сценариев на базе новых шаблонов (RFC, Incident Response).
