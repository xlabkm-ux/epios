# UI Roadmap & Documentation Index

**Документ:** Организационный устав раздела UI Roadmap  
**Статус:** Активен  
**Цель:** Описание структуры и назначения директории `05_ui_roadmap`, которая служит единым центром знаний по развитию пользовательского интерфейса, концепции продукта со стороны фронтенда и планов QA.

## 📁 Структура раздела

Раздел разбит на 5 ключевых документов, каждый из которых отвечает за свой аспект жизненного цикла интерфейса:

### 1. [01_UI_STATE_AND_CHANGELOG.md](./01_UI_STATE_AND_CHANGELOG.md)
**Что внутри:** Живой трекер актуального состояния интерфейса.
- Архитектура текущего UI (demo-shell).
- Матрица состояния компонентов (что работает на моках, а что подключено к реальному API).
- Changelog изменений интерфейса по завершенным спринтам.
*Обновляется после каждого спринта.*

### 2. [02_UI_VISION_AND_WORKFLOW.md](./02_UI_VISION_AND_WORKFLOW.md)
**Что внутри:** Концептуальное продуктовое видение.
- Глобальная дизайн-концепция (Premium UI, Dark Mode).
- User Journeys и Workflows для разных ролей (Reviewer, Author).
- План интеграции с Template Library (динамический UI на основе профилей).
- Подходы к визуализации сложного контекста (Event Sourcing, Графы).

### 3. [03_UI_DEVELOPMENT_PLAN.md](./03_UI_DEVELOPMENT_PLAN.md)
**Что внутри:** Стратегический и тактический план разработки (Roadmap).
- Бэклог интерфейсных фичей за рамками MVP v1.1 (на v1.2, v2.0).
- Трекер фиксации выполненных крупных блоков (что перенесено в production).
- Задачи по техническому долгу UI (рефакторинг, стейт-менеджмент).

### 4. [04_UI_QA_AND_METRICS.md](./04_UI_QA_AND_METRICS.md)
**Что внутри:** Обеспечение качества интерфейса.
- Сценарии UI QA (e2e Playwright).
- Метрики успешности юзабилити (целевое время, количество кликов).
- Реестр обратной связи по итогам Pilot Phase (UX проблемы и их статусы).

### 5. [05_UI_DESIGN_SYSTEM_GUIDE.md](./05_UI_DESIGN_SYSTEM_GUIDE.md)
**Что внутри:** Технические и визуальные стандарты для разработчиков (и AI-агентов).
- Дизайн-токены (цвета, шрифты, отступы).
- Гайдлайны по анимациям (Framer Motion).
- Архитектурные паттерны компонентов и состояний (Loading, Error, Empty).

---
*Примечание: Данный раздел является единым источником правды для развития фронтенда проекта Epistemic OS.*
