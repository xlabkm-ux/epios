# Концепция UI, Рабочие процессы и Видение

## 1. Глобальное видение (Premium UI Vision)

Пользовательский интерфейс Epistemic OS (EpiOS) должен вызывать чувство глубокого профессионализма и технологического превосходства ("Wow-эффект"). 

**Ключевые принципы:**
- **Premium Aesthetics:** Использование глубоких темных тонов (Dark Mode по умолчанию), неоновых акцентов (синий/индиго для нейтральных действий, зеленый/красный для статусов), эффектов glassmorphism (полупрозрачные панели с размытием фона).
- **Живой отклик (Dynamic Feel):** Интерфейс должен ощущаться "живым". Каждый клик, наведение или смена контекста сопровождается плавной микроанимацией (Framer Motion). Никаких резких скачков верстки.
- **Снижение когнитивной нагрузки:** Сложные данные (Readiness-оценки, графы) подаются порционно. Сначала верхнеуровневый статус (Цвет + Иконка), по клику — детализация (аккордеоны).

## 2. Идеальный User Workflow (Ролевые сценарии)

### А. Сценарий Ревьюера (Reviewer Workflow)
1. **Вход:** Ревьюер открывает ADR Review Workspace. Видит общую Mission и статус Readiness (например, 🟡 Needs Review).
2. **Анализ графа:** На `GraphCanvas` визулизируются ключевые узлы (Claims, Evidence, Policies). Красные узлы привлекают внимание к отсутствующим доказательствам.
3. **Оценка источника:** Ревьюер читает исходный документ в `SourcePanel` и, при необходимости, подтверждает или понижает рейтинг доверия в `RatingPanel`.
4. **Ревью патчей:** В `GovernancePanel` ревьюер видит предложенные изменения (Patches). Читает обоснование (Rationale).
5. **Принятие решения:** Ревьюер нажимает "Approve", интерфейс плавно переводит Readiness статус в 🟢 Ready, генерируется финальная версия артефакта.

### Б. Сценарий Автора (Author Workflow)
1. **Загрузка:** Автор загружает черновик ADR.
2. **Ожидание маппинга:** Наблюдает за асинхронным разбором документа в `MappingPanel` (прогресс-бар, логи шагов).
3. **Исправление:** Замечает, что система пометила утверждение как "Overbroad" (Слишком широкое).
4. **Действие:** В `GovernancePanel` автор создает `Patch`, предлагая сузить скоуп решения.

## 3. Развитие: Связь с Template Library Registry

В будущем (v1.2+) EpiOS не будет ограничен только ADR. Внедрение **Template Library** потребует кардинальной гибкости UI:
- **Динамический Layout:** Сайдбар и панели будут генерироваться на основе JSON-схемы шаблона. 
- **Профили оценки (Profiles):** Если профиль требует "Security Check", UI автоматически подгрузит новую панель `SecurityAssessmentPanel`.
- **Селектор Шаблонов:** Появится новый стартовый экран создания Workspace, где пользователь визуально (в виде карточек с превью) выбирает шаблон артефакта (ADR, RFC, Incident Report).

## 4. Развитие: Визуализация Audit & Event Sourcing

Для обеспечения прозрачности решений:
- **Time-machine UI:** Ползунок времени в `GraphCanvas`, позволяющий "отмотать" состояние графа и панелей на момент конкретного события или патча.
- **Traceability View:** Разворачиваемая инспектор-панель для каждого узла, показывающая криптографический хеш изменения, автора и ссылку на исходную строчку в документе.
