# UI Design System & Development Guide

Этот документ описывает технические и визуальные стандарты, которых обязан придерживаться любой разработчик (и AI-агент) при создании или модификации интерфейсов Epistemic OS.

## 1. Цветовая палитра (Dark Mode Premium)

Мы используем кастомную темную тему, основанную на холодных оттенках и строгих акцентах. Использовать ТОЛЬКО CSS-переменные из `index.css`.

- **Backgrounds:**
  - `var(--bg-base)`: Основной фон холста (очень темный, #0a0a0a).
  - `var(--bg-panel)`: Фон панелей (glassmorphism эффект, rgba).
  - `var(--bg-panel-hover)`: Подсветка панели при наведении.
- **Text:**
  - `var(--text-primary)`: Основной текст (белый / светло-серый).
  - `var(--text-secondary)`: Приглушенный текст (заголовки панелей, логи).
- **Accents (Semantic):**
  - `var(--accent-blue)`: Нейтральное действие, инфо (например, кнопка Propose).
  - `var(--accent-green)`: Успех, Ready, Approve.
  - `var(--accent-red)`: Ошибка, Blocked, Overbroad claim.
  - `var(--accent-amber)`: Предупреждение, Needs Review.

## 2. Гайдлайны по анимациям (Framer Motion)

Для создания "Wow-эффекта" мы используем библиотеку `framer-motion`. 
**Правила:**
- **Без резких скачков:** Появление новых элементов (нового патча, модалки) всегда должно быть обернуто в `<motion.div>` с `initial={{ opacity: 0, y: 10 }}` и `animate={{ opacity: 1, y: 0 }}`.
- **Hover эффекты:** Интерактивные карточки должны слегка приподниматься: `whileHover={{ scale: 1.02 }}`.
- **Переходы:** Использовать пружинную физику для плавности: `transition={{ type: "spring", stiffness: 300, damping: 25 }}`.

## 3. Архитектура UI Компонентов

1. **Разделение логики:** Бизнес-логика (API вызовы, маппинг данных) должна жить в хуках (`useApi.ts`, `WorkspaceContext.tsx`). React-компоненты (в папке `components/`) должны быть максимально "глупыми" (отвечать только за рендер).
2. **Стилизация:** Никакого Tailwind! Используем строгий Vanilla CSS. Классы именуем по БЭМ-подобной методологии (например, `panel-container`, `panel-header`, `action-button`).
3. **Отзывчивость (Responsive):** Панели должны использовать Flexbox или CSS Grid. Для отступов использовать `gap` и `padding` в `rem` единицах (например, `1rem`, `0.5rem`).

## 4. Обработка состояний (UI States)

Любой компонент, работающий с данными, должен явно обрабатывать 4 состояния:
- **Idle / Loading:** Пока данные грузятся, показывать красивый скелетон (Skeleton) или пульсирующий индикатор (Pulse dot). Не использовать скучные системные спиннеры.
- **Data (Success):** Рендер полезной нагрузки.
- **Empty:** Если данных нет (например, 0 патчей), показывать стильную заглушку: полупрозрачная иконка и текст "No patches proposed yet". Не оставлять пустые блоки.
- **Error:** Если API вернуло ошибку, показывать красный бейдж с текстом ошибки и кнопкой "Retry". Приложение не должно падать (White Screen of Death). Используйте Error Boundaries.
