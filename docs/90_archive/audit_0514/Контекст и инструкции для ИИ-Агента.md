## **\<HARD\_INVARIANTS\>**

**КРИТИЧЕСКИ ВАЖНО:** Нарушение этих правил приведет к отклонению твоего кода.

1. **NO DIRECT COMMITS:** Ты никогда не коммитишь в main. Все изменения оформляются только через Pull Requests.  
2. **DEPENDENCY RULE:** Слой domain **НЕ ИМЕЕТ ПРАВА** импортировать модули из infrastructure (например, SQL-драйверы, ORM). Домен работает только с чистыми типами и интерфейсами.  
3. **NO COMPLEX MATH:** В MVP v1.1 **ЗАПРЕЩЕНЫ** сложные расчеты "адекватности" (Adequacy Score) от 0 до 100, веса и штрафы.  
   * Использовать только 3 качественные метрики: Evidence Coverage (low/medium/high), Traceability (missing/partial/full), Risk Handling (absent/implicit/explicit).  
4. **HUMAN APPROVAL IS KING:** Никакой расчетный показатель системы не может автоматически разрешить действие (apply patch). Система только *рекомендует* или *блокирует*. Approval всегда ставит человек (Human-in-the-loop).  
5. **TRACE EVERYTHING:** Любая мутация артефакта (ArtifactPatch) **ОБЯЗАНА** сопровождаться созданием события TraceEvent. Патч без причины/обоснования выбрасывает ошибку на уровне домена.  
   \</HARD\_INVARIANTS\>

## **\<ONTOLOGY\_AND\_TYPES\>**

Используй эти термины и базовые типы при генерации кода. Не выдумывай новые сущности.

// CORE DOMAIN VOCABULARY

type MissionId \= string;  
type ArtifactId \= string;

// 1\. Предмет анализа (Что мы ревьюим)  
interface EngineeringDecision {  
  id: ArtifactId;  
  content: string; // Markdown  
  status: 'draft' | 'review' | 'approved';  
}

// 2\. Оценка человеком (Human Rating)  
interface HumanAdequacyRating {  
  missionId: MissionId;  
  subjectRef: ArtifactId;  
  rating: 'reliable' | 'partial' | 'unreliable';  
  rationale: string; // Обоснование оценки  
}

// 3\. Системная оценка (System Assessment)  
interface AdequacyAssessment {  
  evidenceCoverage: 'low' | 'medium' | 'high';  
  traceability: 'missing' | 'partial' | 'full';  
  riskHandling: 'absent' | 'implicit' | 'explicit';  
  readinessBand: 'blocked' | 'needs\_review' | 'usable\_with\_limitations' | 'strong';  
}

// 4\. Предлагаемое изменение  
interface ArtifactPatch {  
  patchId: string;  
  artifactId: ArtifactId;  
  diff: string;  
  reason: string; // Обязательно\!  
  evidenceRefs: string\[\]; // Ссылки на EpistemicNodes или внешние URL  
}

// 5\. Утверждение (Approval)  
interface ApprovalState {  
  patchId: string;  
  approvedByHuman: boolean; // Если false \-\> patch нельзя применить  
  actorId: string;  
}

\</ONTOLOGY\_AND\_TYPES\>
