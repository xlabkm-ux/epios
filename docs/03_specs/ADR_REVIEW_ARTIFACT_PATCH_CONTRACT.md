# ADR Review Artifact Patch Contract

Contract for patching an ADR.

## Patch Lifecycle
- **Reason**: Why the patch is needed.
- **Evidence**: Supporting data.
- **Approval**: Required sign-offs.
- **Trace**: Observability and lineage tracking.
- **Rollback**: Mechanism to revert the patch safely.
