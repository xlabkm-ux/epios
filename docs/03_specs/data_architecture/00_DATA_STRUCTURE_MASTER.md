# 🏛️ Реестр и Структура Данных Epistemic OS (Data Architecture)

Этот раздел описывает каноническую структуру всех данных проекта: от управления пользователями до корпоративной структуры и доменных сущностей.

## 1. Пользователь и Идентификация (Identity)

### Сущность `User`
Центральная сущность участника системы.

| Поле | Тип | Описание |
| :--- | :--- | :--- |
| `uid` | UUID | Глобальный уникальный идентификатор пользователя. |
| `name` | String | Полное имя пользователя. |
| `email` | String | Корпоративный e-mail (первичный логин). |
| `passwordHash` | String | Хеш пароля (Bcrypt/Argon2). |
| `twoFactorSecret` | String | Секрет для генерации кодов (TOTP). |
| `is2FAEnabled` | Boolean | Флаг активности двухфакторной авторизации. |
| `subordination` | Table | Таблица вхождения в корпоративную структуру. |

---

## 2. Корпоративная Структура (Corporate Structure)

Система поддерживает иерархическую структуру клиента, где каждый пользователь может занимать разные роли в разных подразделениях.

### Сущность `OrgUnit` (Элемент структуры)
- `id`: UUID
- `name`: Название (отдел, департамент, проектная группа)
- `parentId`: Ссылка на родительский юнит
- `type`: Тип (`department`, `squad`, `division`)

### Таблица Подчинения (`UserMembership`)
Связывает пользователя с оргструктурой.

| UserUID | OrgUnitID | Role | Permissions |
| :--- | :--- | :--- | :--- |
| `u-123` | `dept-456` | `Lead` | `admin`, `approve` |
| `u-123` | `project-789` | `Contributor` | `propose` |

---

## 3. Доменные данные (Core Domain)

### Эпистемический Граф (Graph)
- `Node`: Узлы знаний (claims, evidence).
- `Edge`: Логические связи (supports, refutes).

### Артефакты Governance
- `Mission`: Контекст задачи.
- `Source`: Первоисточники данных.
- `ArtifactPatch`: Предложенные изменения.
*   `DecisionRecord (ADR)`: Финальный зафиксированный результат.

---

## 4. Безопасность и Аудит

### `AuditRecord`
Журнал всех критических действий.
- `timestamp`: Время события.
- `actorId`: Кто совершил (UID).
- `action`: Что сделано.
- `context`: JSON с деталями (IP, UserAgent, Metadata).

### `RedactionRule`
Правила маскирования.
- `pattern`: Regex.
- `replacement`: Строка замены.
