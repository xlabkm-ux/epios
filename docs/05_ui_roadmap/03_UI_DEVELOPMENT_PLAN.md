# План развития интерфейсов (UI Roadmap)

В данном документе фиксируется план развития интерфейсной части продукта за рамками MVP v1.1.

## 🟢 Выполнено (MVP v1.1)
- [x] Каркас приложения, сайдбар, дизайн-система (Dark Mode).
- [x] Интеграция React Flow для визуализации Epistemic Graph.
- [x] Панели управления данными: Mission, Source, Rating.
- [x] Панели асинхронного маппинга и governance (Readiness, Patches, Voting).
- [x] Интеграция ролевой модели (отключение кнопок для ReadOnly).

---

## 🟡 Горизонт 1: Post-Pilot UX Polish (Ближайшее время)
*Задачи по итогам обратной связи первой волны тестирования.*
- [ ] **Улучшение навигации в графе:** Кнопки центрирования, зумирования на конкретный узел по клику в панели Governance.
- [ ] **Traceability Summary:** Читабельный модальный экран, показывающий человекопонятный лог того, почему готовность (Readiness) изменилась после применения патча.
- [ ] **Responsive Design:** Гарантия корректного отображения панелей на небольших экранах ноутбуков (скролл-контейнеры, схлопывание сайдбара).
- [ ] **Экспорт:** Кнопка визуального скачивания (PDF/Markdown) отрендеренного финального ADR.
- [ ] **Interactive Debug:** Итеративная отладка элементов интерфейса в диалоге с разработчиком (полировка UX "на лету").

---

## 🔵 Горизонт 2: Template Library & Collaborative UI (Среднесрочная перспектива)
*Переход от жесткого ADR-shell к гибкой платформе.*
- [ ] **Стартовый экран (Dashboard):** Список рабочих пространств (Workspaces) пользователя с фильтрами и статусами.
- [ ] **Template Selector:** Интерфейс выбора шаблона (ADR, RFC, Tech Spec) при создании Workspace.
- [ ] **Dynamic Panel Registry:** Архитектурный рефакторинг UI, позволяющий рендерить панели на основе конфигурации профиля шаблона (а не хардкодить их в `ADRReviewWorkspace.tsx`).
- [ ] **Multi-Reviewer Presence:** Индикаторы присутствия (аватарки) других пользователей в текущем Workspace (подобно Figma/Miro).
- [ ] **Inline Comments:** Возможность оставлять текстовые комментарии прямо к узлам графа или строкам в SourcePanel.
- [ ] **Collaborative Debug:** Совместная отладка сложных состояний интерфейса в диалоге с разработчиком.

---

## 🟣 Горизонт 3: Enterprise & Ecosystem (Стратегия)
- [ ] **Admin & RBAC UI:** Реализация интерфейсов для управления ролями и SSO (см. детальный план в `docs/08_user_management_roadmap/05_RBAC_DEVELOPMENT_PLAN.md`).
- [ ] **MCP & Plugin Interfaces:** Зона инжекции UI-виджетов сторонних плагинов (см. `docs/09_ai_and_mcp_integration_roadmap/01_AI_MCP_ROADMAP.md`).
- [ ] **Audit / Time-Machine:** Визуальный интерфейс для Event Sourcing (ползунок времени графа, просмотр цепочки хешей).
- [ ] **External Integrations UI:** Экраны настройки биндингов (вебхуки, OAuth) для GitHub PRs, Slack-уведомлений.
- [ ] **Ecosystem Debug:** Финальная отладка и стабилизация всех интерфейсных модулей в диалоге с разработчиком.

---

## 🛠 Технический долг UI
- [ ] **Перевод стейта на Zustand / Redux:** Текущий `WorkspaceContext` и локальные стейты могут не выдержать нагрузки при добавлении коллаборативного режима реального времени. Необходимо запланировать миграцию.
- [ ] **Storybook:** Внедрение Storybook для независимой разработки и тестирования переиспользуемых UI-компонентов (кнопки, бейджи, модалки).
